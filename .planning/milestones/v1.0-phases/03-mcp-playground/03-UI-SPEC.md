## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps in tool list items, inline badge padding |
| sm | 8px | Tool list item internal padding, form field gaps |
| md | 16px | Column internal padding, card padding, search input padding |
| lg | 24px | Column gaps between the three panels |
| xl | 32px | Page-level content padding (matches existing `p-8`) |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: Search input and tool list items use 40px min-height for comfortable click targets. Column dividers use 1px border.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Tool descriptions, parameter labels, status bar text |
| Label | 12px | 600 | 1.4 | Domain group headers, badge text, muted captions, execution time |
| Heading | 20px | 600 | 1.2 | "MCP Playground" page title, panel headers ("Parameters", "Response") |
| Code | 13px | 400 | 1.6 | JSON viewer text, tool names in agent view, monospace param values |

Font mapping: Body = `text-sm font-normal`, Label = `text-xs font-semibold`, Heading = `text-xl font-semibold`, Code = `text-[13px] font-mono font-normal leading-relaxed`.

## Color

| Role | Value | CSS Variable | Usage |
|------|-------|--------------|-------|
| Dominant (60%) | `hsl(222 47% 8%)` | `--background` | Page background |
| Secondary (30%) | `hsl(222 40% 12%)` | `--card` | Glassmorphism panels (tool selector, param form, JSON viewer) |
| Accent (10%) | `hsl(270 95% 65%)` | `--primary` | See reserved list below |
| Destructive | `hsl(0 72% 51%)` | `--destructive` | Error status badge, failed response border tint |
| Success | `hsl(142 72% 45%)` | `--success` | Success status badge, dry-run success indicator |
| Warning | `hsl(38 92% 50%)` | `--warning` | Dry-run mode status badge |
| Muted foreground | `hsl(222 15% 55%)` | `--muted-foreground` | Inactive tool names, placeholder text, secondary JSON text |

**Accent reserved for:**
- Selected tool highlight in tool selector (left border or background tint)
- "Execute" primary CTA button
- Active toggle state on Human/Agent view switch
- Global dry-run toggle active state
- Ring/focus outlines on interactive elements
- Search input focus border

**JSON syntax highlighting colors** (VS Code dark-inspired):
- Strings: `hsl(142 60% 60%)` (green)
- Numbers: `hsl(210 80% 65%)` (blue)
- Booleans/null: `hsl(270 60% 70%)` (purple/lavender)
- Keys/properties: `hsl(220 15% 75%)` (light grey)
- Brackets/punctuation: `hsl(222 15% 55%)` (muted)

**Glassmorphism panel pattern** (consistent with Phase 1/2):
```
bg-card/60 border border-border/50 backdrop-blur-lg rounded-xl
```
All three columns use this pattern as their outer container.

## Interaction Patterns

### Tool Selection
1. User clicks domain group header to expand/collapse (chevron rotates)
2. All domain groups start expanded (shows tool breadth to judges)
3. User clicks a tool item -- it highlights with accent left-border
4. Parameter form populates with that tool's schema fields
5. Previous form values cleared on tool switch
6. Search input filters tools across all domains in real-time; empty domains auto-hide

### Parameter Form
1. Form fields generated dynamically from tool's Zod schema
2. Smart defaults pre-filled: wallet address for address fields, "1.0" for amounts, `dry_run` controlled by global toggle (hidden from individual form)
3. Address fields validate `0x` prefix + 42 chars on blur
4. Amount fields validate positive number on blur
5. Required fields show red border (`border-destructive`) + helper text on blur if empty
6. Submit button ("Execute Tool") disabled until all required fields valid

### Tool Execution Flow
1. User fills form, clicks "Execute Tool"
2. Button shows loading state: spinner icon + "Executing..." text, disabled
3. Request JSON appears in top of response panel immediately (shows what was sent)
4. Response panel shows skeleton shimmer below status bar
5. On resolve: status bar populates (Success/Error/Dry Run badge + time + HTTP code), response JSON renders with syntax highlighting
6. On error: status bar shows Error badge (red), response JSON shows error body
7. If global dry-run ON: `dry_run: true` injected into every request; status badge shows "Dry Run" (amber) on success
8. If demo mode (no wallet) + write tool: force dry-run, show demo mode message below status bar

### Human/Agent View Toggle
1. Default: Agent view (snake_case names, raw JSON, raw params)
2. Toggle to Human view: tool names become friendly labels, param labels become readable, response panel adds formatted summary above raw JSON
3. Agent view JSON is always visible (Human view adds summary, does not hide raw)
4. Toggle state persists during session (not across page navigation)

### JSON Viewer Interactions
- Click disclosure triangle to collapse/expand nested JSON nodes
- Click copy button to copy full JSON to clipboard (shows "Copied" confirmation)
- Click download button to save as `.json` file (filename: `{tool_name}-response.json`)
- Click word-wrap toggle to enable/disable wrapping long lines
- JSON nodes start fully expanded (shows rich content to judges)

### Loading States
- Response panel: skeleton shimmer in code-block shape (`rounded-lg h-[200px]`)
- Execute button: inline spinner + "Executing..." text, disabled state
- Tool selector on initial load: skeleton shimmer for tool list items

### Error States
- Inline within the response panel -- never full-page errors
- Error badge (red) in status bar + error response body in JSON viewer
- If bridge is unreachable: show "Unable to connect to MCP bridge. Ensure the server is running." in response panel with muted error styling
