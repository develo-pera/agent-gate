**1. Create `packages/app/src/lib/hooks/use-playground.ts`:**

A "use client" hook managing all playground state. DO NOT use `useMcpAction` — use direct `fetch()` because tool name is dynamic.

State:
- `selectedTool: string | null` — currently selected tool name
- `formValues: Record<string, unknown>` — current form field values
- `globalDryRun: boolean` — starts `true`
- `isHumanView: boolean` — starts `false` (agent view default)
- `searchQuery: string` — tool filter text
- `request: Record<string, unknown> | null` — last request body sent
- `response: Record<string, unknown> | null` — last response body received
- `status: { code: number; time: number; mode: "success" | "error" | "dry_run" } | null`
- `isExecuting: boolean`

Behaviors:
- `selectTool(name: string)` — sets selectedTool, clears formValues/request/response/status, then pre-fills smart defaults from the tool's schema params: for `isAddress` params use `activeAddress` from `useApp()`, for `isAmount` params use `"1.0"`, for params with `default` use that default
- `setFormValue(key: string, value: unknown)` — updates single form field
- `setGlobalDryRun(v: boolean)` — updates globalDryRun
- `setIsHumanView(v: boolean)` — updates isHumanView
- `setSearchQuery(v: string)` — updates searchQuery
- `executeTool()` — async function that:
  1. Gets the selected tool's schema from `TOOL_SCHEMAS`
  2. Builds request body: `{ ...formValues, wallet_address: activeAddress }`
  3. If tool `hasWriteEffect` and (`globalDryRun || isDemo`): adds `dry_run: true` to body
  4. Sets `request` to the body object
  5. Sets `isExecuting = true`
  6. Calls `fetch(/api/mcp/${selectedTool}, { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify(requestBody) })`
  7. Records `performance.now()` before and after to get execution time in ms
  8. Parses response JSON
  9. Sets `response` to the data
  10. Sets `status`: code from `res.status`, time in ms (rounded), mode = `res.ok ? (dryRunWasActive ? "dry_run" : "success") : "error"`
  11. Sets `isExecuting = false`
  12. On catch: sets response to `{ error: message }`, status to `{ code: 0, time: 0, mode: "error" }`, isExecuting false

Import `useApp` from `@/providers/app-provider`, `TOOL_SCHEMAS` from `@/lib/tool-schemas`.
