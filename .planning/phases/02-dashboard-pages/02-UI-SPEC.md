## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, form field gaps |
| md | 16px | Default card padding, element spacing |
| lg | 24px | Card internal padding, section gaps |
| xl | 32px | Page-level content padding (matches existing `p-8`) |
| 2xl | 48px | Major section breaks between page zones |
| 3xl | 64px | Not used this phase |

Exceptions: Touch targets minimum 44px height (already established in sidebar nav items via `min-h-[44px]`). Form submit buttons use 44px min-height.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Table cells, form labels, card body text |
| Label | 12px | 600 | 1.4 | Stat labels, badge text, muted captions |
| Heading | 20px | 600 | 1.2 | Page titles, section headings, card titles |
| Display | 32px | 600 | 1.1 | Hero metrics (APR number, vault total, health score) |

Tailwind classes: `text-sm` (14px body), `text-xs` (12px label), `text-xl` (20px heading), `text-3xl` (32px display). Weights: `font-normal` (400), `font-semibold` (600). No other weights.

## Color

| Role | Value | CSS Variable | Usage |
|------|-------|--------------|-------|
| Dominant (60%) | `hsl(222 47% 8%)` | `--background` | Page background |
| Secondary (30%) | `hsl(222 40% 12%)` | `--card` | Glassmorphism cards, sidebar |
| Accent (10%) | `hsl(270 95% 65%)` | `--primary` | See reserved list below |
| Destructive | `hsl(0 72% 51%)` | `--destructive` | Revoke button, error badges, health score red zone |
| Success | `hsl(142 72% 45%)` | `--success` | Dry-run success badge, health score green zone, yield positive indicator |
| Warning | `hsl(38 92% 50%)` | `--warning` | Health score yellow zone, approaching limits |
| Muted foreground | `hsl(222 15% 55%)` | `--muted-foreground` | Secondary text, inactive labels |

**Accent reserved for:**
- Hero APR glow effect (`text-shadow: 0 0 20px hsl(270 95% 65% / 0.5)`)
- Active nav indicator border (existing)
- Primary CTA buttons (Deposit, Create Delegation)
- Donut chart principal segment fill
- Ring/focus outlines on interactive elements
- Chart-1 color for primary data series

**Glassmorphism card pattern** (established in Phase 1):
```
bg-card/60 border border-border/50 backdrop-blur-lg rounded-xl
```
All dashboard cards use this pattern. Do not use opaque `bg-card` for content cards.

**Chart colors** (use CSS variables in order):
- `--chart-1` (purple/primary): Principal balance
- `--chart-2` (green/success): Yield balance
- `--chart-3` (amber/warning): Warning indicators
- `--chart-4` (muted): Inactive/empty segments
- `--chart-5` (foreground): Labels on dark backgrounds

## Interaction Patterns

### Loading States
- All data cards show skeleton shimmer matching the final layout shape
- Skeletons use `rounded-xl` matching card radius, `h-[200px]` for chart area, `h-[20px] w-[120px]` for stat values
- Loading appears immediately on mount; data replaces skeleton on resolve

### Error States
- Inline within the affected card/section -- never full-page errors
- Show muted error icon (AlertCircle from lucide, 24px), error copy, and "Retry" button (secondary variant)
- Error card uses same glassmorphism pattern with `border-destructive/30` border tint

### Dry-Run Flow
1. User fills form, enables dry-run toggle (default OFF)
2. User clicks submit button
3. If dry-run ON: POST to `/api/mcp/[tool]` with `dry_run: true`, show `dry-run-result` below form
4. If dry-run OFF and wallet connected: POST to MCP bridge for real execution
5. If dry-run OFF and demo mode: Force dry-run, show demo mode message
6. Result card stays visible until user dismisses (X button) or submits again

### Delegation View Toggle
- shadcn `tabs` component with two options: "Cards" and "Table"
- Default: "Cards"
- Toggle sits in the page header row, right-aligned next to "Create Delegation" button
- View preference does not persist across navigation

### Destructive Actions
- Revoke button uses `variant="destructive"` on the button
- Clicking Revoke opens a confirmation dialog (shadcn `dialog`) with the revoke confirmation copy
- Dialog has "Cancel" (secondary) and "Revoke" (destructive) buttons
- No confirmation needed for deposit/withdraw (dry-run toggle serves this purpose)

### Form Validation
- Required fields show red border (`border-destructive`) and helper text on blur if empty
- Address fields validate hex format (`0x` prefix, 42 chars) on blur
- Amount fields validate positive number, no negative, no non-numeric
- Submit button disabled until all required fields valid
