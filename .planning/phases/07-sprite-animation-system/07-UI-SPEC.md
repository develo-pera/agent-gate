## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot margin, inline padding |
| sm | 8px | Hover card internal padding between rows, sprite scene padding |
| md | 16px | Hover card outer padding, scene container padding |
| lg | 24px | Scene top/bottom margin from page content |
| xl | 32px | Not used this phase |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: Sprite frame size is 32x32px native grid, rendered at 96px (3x scale) -- not a spacing token, a content dimension. Scene banner height is 250px -- a layout dimension, not from the spacing scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.2 |

Phase 7 typography is limited to the hover detail card. Only 3 sizes needed:
- **Heading (16px/600):** Agent name in hover card
- **Body (14px/400):** Current action text in hover card
- **Label (12px/400):** Truncated address, status label in hover card. Visually distinguished from body by smaller size, `font-mono`, and `muted-foreground` color -- no separate weight needed.

| Status | Color Variable | HSL | Dot Usage |
|--------|---------------|-----|-----------|
| active | `var(--success)` | hsl(142 72% 45%) | Green pulsing dot in hover card |

Accent reserved for: Phase 8 agent cards and dashboard chrome. Phase 7 hover cards use status colors only -- no hot-pink accent in sprite UI.

### Sprite Body Colors (address-derived)

Sprite robot body colors are deterministically generated from the agent's Ethereum address hash. The palette rotates through 8 hues at fixed saturation/lightness to guarantee visibility on the dark background:

| Slot | Hue | HSL Template |
|------|-----|-------------|
| 0 | 0 (red) | hsl(0, 70%, 60%) |
| 1 | 45 (orange) | hsl(45, 70%, 60%) |
| 2 | 90 (lime) | hsl(90, 70%, 60%) |
| 3 | 135 (green) | hsl(135, 70%, 60%) |
| 4 | 180 (cyan) | hsl(180, 70%, 60%) |
| 5 | 225 (blue) | hsl(225, 70%, 60%) |
| 6 | 270 (purple) | hsl(270, 70%, 60%) |
| 7 | 315 (pink) | hsl(315, 70%, 60%) |

Selection: `parseInt(address.slice(2, 4), 16) % 8` to pick the hue slot. Applied via CSS custom property `--sprite-body` on the SVG.

## Sprite Animation Contract

### SVG Sprite Sheet Structure

| Property | Value |
|----------|-------|
| Frame size | 32x32px native |
| Render scale | 3x (96x96px on screen) |
| Frames per row | 4 |
| Rows | 3 (idle=row 0, walk=row 1, work=row 2) |
| Total frames | 12 per character |
| Sheet dimensions | 128x96px native (384x288px at 3x) |
| Rendering | `image-rendering: pixelated` on container |

### CSS Animation Keyframes

| Animation | Row | Frames | Duration | Timing | Iteration |
|-----------|-----|--------|----------|--------|-----------|
| sprite-idle | 0 (y=0) | 4 | 1s | `steps(4)` | infinite |
| sprite-walk | 1 (y=-32px native, -96px rendered) | 4 | 1s | `steps(4)` | infinite |
| sprite-work | 2 (y=-64px native, -192px rendered) | 4 | 1s | `steps(4)` | infinite |

Frame advancement is purely CSS -- zero JS timers, zero React state for frame index. The `background-position-x` animates from `0` to `-384px` (4 frames x 96px rendered) using `steps(4)`. Row selection uses `background-position-y`.

### State-Animation Mapping

| AgentStatus | Animation | Movement |
|-------------|-----------|----------|
| `active` | sprite-work | Stationary (stays in place) |
| `idle` | Alternates: sprite-idle (pause 2-5s) then sprite-walk (move to random point) | Random wandering |
| `registered` | sprite-idle | Stationary (no wandering) |

Default when no `status` prop: `idle` with random wandering.

### Scene Movement

| Property | Value |
|----------|-------|
| Scene container | Full-width banner, 250px height |
| Position method | CSS `transition: left 0.8s ease-in-out, top 0.8s ease-in-out` on absolutely-positioned sprites |
| Direction facing | `transform: scaleX(-1)` when moving left |
| Idle pause | Random 2000-5000ms between wander destinations |
| Wander boundary | 48px inset from scene edges (prevents sprite clipping) |
| Z-indexing | Sprites at `z-index: 1`, hover card at `z-index: 10` |

## Hover Detail Card

| Property | Value |
|----------|-------|
| Trigger | `mouseenter` on sprite container |
| Dismiss | `mouseleave` from sprite container |
| Position | Anchored above sprite, centered horizontally |
| Offset | 8px gap above sprite top edge |
| Animation | `opacity 0 -> 1` over 150ms ease-out on enter, 100ms ease-in on leave |
| Width | 200px fixed |
| Background | `var(--card)` with `ring-1 ring-foreground/10` (matches Card component) |
| Border radius | `var(--radius)` (0.75rem) |
| Padding | 16px |

### Card Content Layout

```
+----------------------------------+
| [status dot] Agent Name     16px |  -- heading weight 600
| 0xAbCd...1234               12px |  -- label, font-mono, muted-foreground
| Calling transfer()...       14px |  -- body, foreground
+----------------------------------+
```

| Row | Content | Style |
|-----|---------|-------|
| 1 | Status dot (8x8px rounded-full) + agent name | dot: status color, name: 16px/600 foreground |
| 2 | Truncated address (use AddressDisplay pattern: first 6 + last 4 chars) | 12px/400 muted-foreground, font-mono |
| 3 | Current action text (or "Idle" / "Standing by") | 14px/400 foreground |

Row gap: 4px. Status dot margin-right: 4px.

### Hover Behavior

- Sprite pauses movement on hover (cancels pending wander, holds position)
- Sprite switches to idle animation while hovered (regardless of prior state)
- On mouseleave: card fades out, sprite resumes prior state and movement pattern
