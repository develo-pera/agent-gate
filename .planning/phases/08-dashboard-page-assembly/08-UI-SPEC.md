Accent is NOT used for: agent cards, activity rows, status indicators, or general interactive elements.

## Layout Specification

### Page Structure (top to bottom)

```
+------------------------------------------------------------------+
| h1 "Live Agents"   [3 agents]  [47 events]  [2 active]          |
|                                                   [Demo Mode] *  |
+------------------------------------------------------------------+
| SpriteScene (250px height, full width, bg-secondary rounded-lg)  |
| Agents wander with status-driven animations                      |
+------------------------------------------------------------------+
| AgentCardRow (flex row, gap-16px, flex-wrap)                     |
| [Card: Hackaclaw] [Card: Merkel] [Card: Agent3] ...  [All]      |
+------------------------------------------------------------------+
| ActivityFeed (max-h-[400px] overflow-y-auto, bg-card rounded-lg) |
| | timestamp | agent | tool | status badge |        expandable   |
| | timestamp | agent | tool | status badge |        expandable   |
| | ...                                                            |
| [N new events] badge (sticky top, only if user scrolled down)    |
+------------------------------------------------------------------+
```

* "Demo Mode" chip appears only while demo is running.

### Agent Card Layout

```
+-------------------------------------+
| [dot] Agent Name          status    |
| 0x1a2b...3c4d (mono, muted)        |
| Last: checkVaultHealth (muted)      |
+-------------------------------------+
```

- Width: min-w-[200px], flex-1, max-w-[280px]
- Padding: 16px
- Border: 1px `--border`, selected state adds `ring-1 ring-primary/50`
- Cursor: pointer (filterable)
- Status dot: 8px circle, pulsing animation for "active" status (CSS `animate-pulse`)

### Activity Row Layout

```
+-------------------------------------------------------------------+
| 12:34:05  Hackaclaw  checkVaultHealth           [success]    [v]  |
+-------------------------------------------------------------------+
| (expanded) params: { vault: "0x..." }                             |
|            result: { health: 0.95, ... }                          |
+-------------------------------------------------------------------+
```

- Row padding: 12px 16px
- Timestamp: 12px mono, muted-foreground, fixed width 64px
- Agent name: 14px semibold, foreground
- Tool name: 14px regular, foreground, `font-mono`
- Chain icon: `Link` from lucide, 14px, shown only for on-chain tx events, placed before tx hash
- Status badge: Badge component with variant mapping: success=default(green bg), pending=secondary, error=destructive
- Expand chevron: `ChevronDown` 16px, rotates 180deg on expand
- New-event glow: `ring-1 ring-primary/30` with 2s fade-out via CSS transition
- Expanded detail: `bg-secondary` panel below row, JSON in `pre` with mono font, 12px, max-h-[160px] overflow-auto
- Row separator: `border-b border-border` between rows

## Interaction Contracts

### Agent Card Filtering
- Click card: set `filterAgent` to that agent's address, card gets selected ring
- Click selected card again: clear filter (show all)
- An "All" pseudo-card (or button) at the end of the row clears the filter
- SSE stream is NOT filtered; filtering is client-side on the rendered list

### Activity Feed Auto-Scroll
- Default: new events prepend to top, container stays scrolled to top
- If user has scrolled down (scrollTop > 48px): stop auto-scrolling, show sticky "N new events" badge at container top
- Click "N new events" badge: scroll to top, resume auto-scroll, clear badge

### Activity Row Expand/Collapse
- Click anywhere on the row to toggle expanded detail panel
- Only one row expanded at a time (accordion behavior)
- Expand animation: CSS `max-height` transition from 0 to auto (use `grid-template-rows: 0fr -> 1fr` pattern)

### Demo Mode
- Empty state (0 events, 0 agents or all registered with no activity): show centered empty illustration area with "Run Demo" button
- Click "Run Demo": drip 10-15 seed events at 2-3 second intervals using `setInterval`
- While dripping: "Demo Mode" chip visible next to page title, "Run Demo" button disabled
- Demo events use real agent names from `/api/agents` response
- Sprite scene agents transition status to match drip events (idle -> active -> idle)
- After drip completes: chip remains, events stay in feed, user can interact normally

### SSE Connection
- Connect on page mount, disconnect on unmount
- Reconnect on error with 3-second backoff
- Send `Last-Event-ID` header on reconnect for replay (Phase 6 supports this)
- Merge SSE events with initial REST fetch (deduplicate by event ID)

## Animation Contracts

| Animation | Trigger | CSS | Duration |
|-----------|---------|-----|----------|
| Status dot pulse | Agent status = "active" | `animate-pulse` (Tailwind built-in) | 2s infinite |
| New event glow | Event arrives via SSE | `ring-1 ring-primary/30` -> `ring-transparent` | 2s ease-out |
| Row expand | Click activity row | `grid-template-rows: 0fr -> 1fr` | 200ms ease |
| Chevron rotate | Row expand/collapse | `rotate-180` | 200ms ease |
| Slide-in | New event prepends to feed | `translateY(-8px) opacity-0` -> `translateY(0) opacity-1` | 300ms ease-out |
| Demo chip appear | Demo mode starts | `opacity-0 -> opacity-1` | 150ms ease |

No JavaScript animation libraries. All CSS transitions/animations only. Sprite animations handled by Phase 7 CSS (steps() keyframes).

## Accessibility Notes

- Agent cards: `role="button"`, `aria-pressed` for selected state, keyboard navigable with Enter/Space
- Activity rows: `role="button"`, `aria-expanded` for expand state
- "N new events" badge: `role="status"`, `aria-live="polite"`
- Status dots: `aria-label` with status text (e.g., "Status: active")
- Demo mode chip: decorative (no special ARIA needed, visible text sufficient)
- Activity feed container: `role="log"`, `aria-label="Activity timeline"`
- Color is never the sole indicator: status dots always paired with text labels in badges
